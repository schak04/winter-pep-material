Jenkins vs GitHub Actions vs GitLab ci
--------------------------------------------------
Jenkins (Jenkinsfile):
Jenkins is an automation server used to build CI/CD pipelines.

Uses Jenkinsfile (written in Groovy or Declarative syntax)
Primarily self-hosted, but can also run in cloud (managed Jenkins services exist)
Very powerful but needs setup & maintenance
In simple term:
“Jenkins is a customizable CI/CD tool where we manage everything ourselves.”
-------------------------------------------------
GitHub Actions (YAML Workflows):-
GitHub Actions is a CI/CD tool inside GitHub.

Uses YAML workflow files
Fully integrated with GitHub
Works best only with GitHub repositories (tight coupling)
Easy to use, minimal setup

In simple term:
“GitHub Actions is an easy CI/CD tool built directly into GitHub.”
----------------------------------------------------
GitLab CI/CD (.gitlab-ci.yml):
GitLab CI/CD is built into GitLab platform.

Uses .gitlab-ci.yml
Pipelines → Stages → Jobs (very structured flow)
Can be cloud or self-hosted

In simple term:
“GitLab CI/CD is a structured pipeline system built into GitLab.”
-----------------------------------------------------
📊 Key Differences:-

| Feature                    | Jenkins (Jenkinsfile)             | GitHub Actions (YAML)       | GitLab CI/CD (.gitlab-ci.yml) |
| -------------------------- | --------------------------------- | --------------------------- | ----------------------------- |
| **Hosting**                | Self-hosted (usually)             | Cloud-hosted (GitHub)       | Cloud + Self-hosted           |
| **Configuration Language** | Groovy / Declarative              | YAML                        | YAML                          |
| **Setup Effort**           | High (manual setup)               | Low (ready to use)          | Low to Medium                 |
| **Integration**            | Works with any repo (via plugins) | Native to GitHub            | Native to GitLab              |
| **Structure**              | Pipeline → Stages → Steps         | Workflow → Jobs → Steps     | Stages → Jobs                 |
| **Execution Machine**      | Jenkins Agents                    | GitHub Runners              | GitLab Runners                |
| **Flexibility**            | Very high (custom plugins)        |High, but less than Jenkins  | Moderate                      |
| **Maintenance**            | You manage everything             | GitHub manages infra        | GitLab manages (or self-host) |
| **Best For**               | Complex enterprise pipelines      | Quick & simple CI/CD        | Structured DevOps pipelines   |
|*Where pipeline lives**     | Separate server                   | Inside repo(.github/workflows)| Inside repo(.gitlab-ci.yml)   |

Summary:-
All three do CI/CD, but Jenkins gives full control, GitHub Actions is easiest, and GitLab is structured and balanced.
