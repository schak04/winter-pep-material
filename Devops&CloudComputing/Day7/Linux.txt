Why Linux is Important in DevOps:-

(a)Server Environment:-
Most production servers run on Linux.

(b)Cloud Platforms:-
AWS, Azure, and Google Cloud mainly use Linux-based systems.

(c)Automation Friendly:-
Linux supports shell scripting (Bash), which helps automate tasks.

(e)Security & Stability:-
Linux is secure, stable, and efficient for production environments.

## 1. Linux Architecture
Linux architecture is divided into the following main layers:

1. Hardware
   * CPU
   * RAM
   * Disk
   * Input/Output devices
   Everything in Linux ultimately runs on this hardware.

2. Kernel

   * Core of the Linux operating system/ heart of Linux
   * Manages hardware resources
   * Handles memory management, process management, device drivers, and system calls
   *It manages communication between software and hardware.

   Ex:-When you run a command, the kernel tells the CPU and memory what to do.

3. Shell

   * Interface between user and kernel
   * It interprets commands entered by the user.
   * Examples: Bash,Zsh, Sh

   When user types any commnad ex- ls shell send this request to kernel

   Note:-Some commna programs used by the users are-->
   vi → text editor
   grep → search text
   cd → change directory
   date → show system date
   compilers / applications
   Multiple users can run applications at the same time.

4. System Utilities

   * Tools and commands used to perform operations
   * Example: ls, cp, mv, ps, top

5. Applications

   * User-installed programs like editors, browsers, servers

In simple terms:
User → Application/Command → Shell → Kernel → Hardware
