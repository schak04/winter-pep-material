What are CSS Selectors?
CSS selectors are patterns used to "select" or target the HTML elements you want to style. Think of them as a way to tell CSS "which elements should I apply these styles to?"

Types of Selectors:

1. ELEMENT SELECTOR
   - Targets HTML tags directly
   - Example: p { color: blue; } targets all <p> tags
   - Used when you want to style all instances of an element

2. CLASS SELECTOR
   - Starts with a dot (.)
   - Example: .header { background-color: navy; }
   - Can be reused multiple times on the same page
   - Used for styling multiple different elements the same way

3. ID SELECTOR
   - Starts with a hash (#)
   - Example: #navbar { position: fixed; }
   - Should be unique on a page (use ID for one element only)
   - Used for unique, specific elements

KEY POINT: Classes vs IDs
â†’ Classes are preferred over IDs for styling
â†’ Using classes improves scalability and maintenance
â†’ Reserve IDs for specific purposes like JavaScript interactions

What is Specificity?
Specificity is a system that CSS uses to determine which style rules should be applied when multiple rules target the same element. Higher specificity wins!

Specificity Order (from highest to lowest):
  1. Inline styles (style attribute in HTML)
     Example: <p style="color: red;">
  2. ID selectors (#myId)
  3. Class selectors (.myClass)
  4. Element selectors (p, div, etc.)

Example:
  <p id="main" class="text" style="color: green;">Hello</p>
  
  #main { color: blue; }        â† ID (higher specificity)
  .text { color: red; }         â† Class (lower specificity)
  p { color: black; }           â† Element (lowest specificity)
  
  Result: Text will be GREEN (inline style wins!)

Ways to Define Colors in CSS:

1. COLOR NAMES
   Example: red, blue, green, orange, etc.
   Simple but limited options

2. HEX VALUES (Hexadecimal)
   Example: #FF5733 (Red), #0000FF (Blue)
   Format: #RRGGBB (Red, Green, Blue values)
   Most commonly used by designers

3. RGB VALUES (Red, Green, Blue)
   Example: rgb(255, 87, 51) produces orange
   Format: rgb(0-255, 0-255, 0-255)
   Good for dynamic colors with variables

Tip: Consistent color usage improves design clarity and makes your website look professional

Font Properties:

1. font-family - Sets the typeface
   Example: font-family: Arial, Helvetica, sans-serif;
   You can provide multiple fonts as fallback
   If Arial isn't available, browser tries Helvetica, then sans-serif

2. font-size - Controls text size
   Example: font-size: 16px; or font-size: 1.5em;

3. font-weight - Controls text thickness
   Example: font-weight: bold; (or 700)
   Can be: normal (400), bold (700), etc.

Tip: Readable fonts improve user experience. Use 14-16px as standard body text size

Why Learn Units?
CSS requires units when defining sizes and spacing. Different units work better in different situations.

Types of Units:

1. PX (Pixels) - FIXED UNIT
   Example: width: 200px;
   Always the same size, regardless of screen size
   Good for: precise measurements, borders

2. % (Percentage) - RELATIVE UNIT
   Example: width: 50%;
   Relative to parent element's size
   Good for: responsive design, flexible layouts

3. EM - RELATIVE UNIT
   Example: font-size: 1.5em;
   Relative to parent font size
   If parent is 16px, 1.5em = 24px

4. REM (Root EM) - RELATIVE UNIT
   Example: margin: 2rem;
   Relative to ROOT (html) font size
   More predictable than em

ğŸ’¡ KEY POINT: Relative units (%, em, rem) help with responsive design
              Your website will adapt better to different screen sizes

What is the Box Model?
Every HTML element is treated as a rectangular box. The Box Model explains how an element's total size is calculated by combining several components.

The Box Model Components (from inside out):

1. CONTENT
   The actual text, images, or other content inside the element

2. PADDING
   Space INSIDE the element, between content and border
   Example: padding: 20px; adds 20px space around content

3. BORDER
   The line that surrounds the padding and content
   Example: border: 2px solid black;

4. MARGIN
   Space OUTSIDE the element
   Creates distance between this element and other elements
   Example: margin: 15px; adds 15px space outside

What is Display?
The display property controls HOW an element appears on the page and how other elements flow around it.

Display Values:

1. BLOCK
   - Takes up full available width
   - Starts on a new line
   - Examples: <div>, <p>, <h1>, <header>
   - Example: div { display: block; }

2. INLINE
   - Takes only the space needed
   - Flows within text (doesn't start new line)
   - Examples: <span>, <a>, <button> (default)
   - Example: span { display: inline; }
   - Note: Cannot set width/height for inline elements

3. INLINE-BLOCK
   - Flows inline like text BUT allows width and height
   - Best of both worlds!
   - Example: img { display: inline-block; }
   - Can be useful for navigation menus

4. NONE
   - Hides the element completely
   - Takes up no space on the page
   - Example: .hidden { display: none; }
   - Different from visibility: hidden (hides but takes space)

What is Flexbox?
Flexbox is a modern CSS layout tool that makes it easy to arrange items in one direction (rows or columns). It handles alignment and spacing automatically!

Key Concepts:

1. Activate Flexbox with: display: flex;

2. Flex Direction
   - row: items flow left to right (default)
   - column: items flow top to bottom
   Example: flex-direction: row;

3. Justify-content (horizontal alignment)
   - flex-start: align to the start
   - flex-end: align to the end
   - center: center items
   - space-between: space items evenly
   - space-around: space with margins
   Example: justify-content: center;

4. Align-items (vertical alignment)
   - flex-start: align to top
   - flex-end: align to bottom
   - center: center items vertically
   Example: align-items: center;

Why Flexbox?
â†’ Flexbox aligns items easily
â†’ Flexbox distributes space efficiently
â†’ Flexbox is preferred over older float methods
â†’ Makes responsive design much simpler
