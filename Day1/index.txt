ğŸ“Œ SECTION 7: HTML DOCUMENT STRUCTURE
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
Every HTML page has a basic structure. Here's the template:

```html
<!DOCTYPE html>
<html>
  <head>
    <!-- Metadata goes here -->
    <title>Page Title</title>
  </head>
  <body>
    <!-- Visible content goes here -->
    <h1>Hello World!</h1>
  </body>
</html>
```

Let's understand each part:

1. <!DOCTYPE html>
   - Tells browser this is an HTML5 document
   - Must be the first line
   - Defines HTML version
   - Without it, browser may behave unexpectedly

2. <html> tag
   - Root element of the entire page
   - Everything goes inside this tag
   - Wraps the entire document

3. <head> section
   - Contains METADATA (information about the page)
   - NOT visible on the webpage
   - Important for browsers and search engines
   
   What goes in <head>:
   - <title>: Page title (shown in browser tab)
   - <meta>: Character encoding, viewport, etc.
   - <link>: CSS files, fonts
   - <script>: JavaScript files

4. <body> section
   - Contains all VISIBLE content
   - Everything the user sees on the page
   - Headings, paragraphs, images, buttons, etc.

What are Semantic Tags?
Semantic tags describe the MEANING of the content they contain. They tell both browsers and developers what role each section plays.

Why Use Semantic HTML?
â†’ Improves readability (easier to understand code)
â†’ Improves accessibility (screen readers understand better)
â†’ Better for SEO (search engines rank better)
â†’ Modern best practice (preferred over <div> everywhere)

Common Semantic Tags:

1. <header>
   Represents the top section of a page
   Usually contains: logo, navigation, company name
   ```html
   <header>
     <h1>My Website</h1>
     <p>Welcome to my site!</p>
   </header>
   ```

2. <nav>
   Represents navigation links
   Contains links to different pages
   ```html
   <nav>
     <a href="#home">Home</a>
     <a href="#about">About</a>
     <a href="#contact">Contact</a>
   </nav>
   ```

3. <section>
   Groups related content together
   Represents a thematic grouping
   ```html
   <section>
     <h2>Features</h2>
     <p>Our amazing features...</p>
   </section>
   ```

4. <article>
   Represents independent, self-contained content
   Example: Blog posts, news articles, reviews
   ```html
   <article>
     <h2>How to Learn HTML</h2>
     <p>HTML is easy to learn...</p>
   </article>
   ```

5. <footer>
   Represents the bottom section of a page
   Usually contains: copyright, links, contact info
   ```html
   <footer>
     <p>&copy; 2025 My Website. All rights reserved.</p>
   </footer>
   ```

   When to use:
   âœ“ Non-sensitive queries (search, filtering)
   âœ“ Small amounts of data

   âœ— Data visible to anyone looking at screen
   âœ— Limited data size
   âœ— Not secure for passwords

2. POST METHOD
   ```html
   <form method="POST" action="/login">
   ```
   
   How it works:
   - Sends data in the request body
   - Data NOT visible in URL
   - Hidden from casual observation
   
   When to use:
   âœ“ Sensitive data (passwords, credit cards)
   âœ“ Large amounts of data
   âœ“ Permanent changes (creating accounts)
   âœ“ Private information
   
   Advantages:
   âœ“ More secure
   âœ“ No limit on data size
   âœ“ Data not shown in URL
