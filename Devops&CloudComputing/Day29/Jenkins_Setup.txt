Local setup on Windows using Docker
-------------------------------------------------
Step 1: Make sure Docker is running
Open Docker Desktop and confirm Docker is active.

Step 2: Make sure Docker is using Linux containers
This is important because the Jenkins project provides a Linux container image, not a Windows container image, for this flow.

Step 3: Open terminal

Use:
Command Prompt, or
PowerShell, or
VS Code terminal 

Open your folder (e.g., jenkins-setup)--> for vs code

Step 4: Create Docker network
command:-docker network create jenkins
This creates a dedicated Docker network for Jenkins and related containers. 
The official Docker install guide uses this as the first step.

Step 5: Start Docker-in-Docker helper container
Run this on Windows:
```
docker run --name jenkins-docker --rm --detach --privileged --network jenkins --network-alias docker --env DOCKER_TLS_CERTDIR=/certs --volume jenkins-docker-certs:/certs/client --volume jenkins-data:/var/jenkins_home --publish 2376:2376 docker:dind
  ```
  Jenkins needs to run Docker commands (build images, run containers)
  This container acts like:Docker inside Docker helper
  This container lets Jenkins trigger Docker commands during builds in a clean tutorial-style setup. 
  The official guide uses this pattern for Jenkins + Docker workflows.

  Step 6: Create a Dockerfile
  Create a new empty folder, for example:
  jenkins-setup
  Inside it, create a file named Dockerfile with this content:
  ```
  FROM jenkins/jenkins:2.541.3-jdk21
USER root
RUN apt-get update && apt-get install -y lsb-release
RUN curl -fsSLo /usr/share/keyrings/docker-archive-keyring.asc \
  https://download.docker.com/linux/debian/gpg
RUN echo "deb [arch=$(dpkg --print-architecture) \
  signed-by=/usr/share/keyrings/docker-archive-keyring.asc] \
  https://download.docker.com/linux/debian \
  $(lsb_release -cs) stable" > /etc/apt/sources.list.d/docker.list
RUN apt-get update && apt-get install -y docker-ce-cli
USER jenkins
RUN jenkins-plugin-cli --plugins "blueocean docker-workflow json-path-api"
```
Run this command in powershell:-
docker build -t myjenkins-blueocean:2.541.3-1 .

This is the official pattern Jenkins documents for building a custom Jenkins image with Docker CLI and common plugins.

Step 7: Build your Jenkins image
In that folder, run:
command:-docker build -t myjenkins-blueocean:2.541.3-1 .
Converts your Dockerfile → usable image

Step 8: Start Jenkins container
Run:
```
docker run --name jenkins-blueocean --restart=on-failure --detach `
  --network jenkins --env DOCKER_HOST=tcp://docker:2376 `
  --env DOCKER_CERT_PATH=/certs/client --env DOCKER_TLS_VERIFY=1 `
  --volume jenkins-data:/var/jenkins_home `
  --volume jenkins-docker-certs:/certs/client:ro `
  --publish 8080:8080 --publish 50000:50000 `
  myjenkins-blueocean:2.541.3-1
  ```
This launches Jenkins and exposes:
8080 for browser UI
50000 for agent communication if needed later

Step 9: Open Jenkins in browser
Go to:http://localhost:8080
The Jenkins setup wizard starts there. 
Jenkins documentation says the first-time wizard unlocks Jenkins, installs plugins, and creates the first admin user.

Step 10: Get initial admin password
Run:
docker exec jenkins-blueocean cat /var/jenkins_home/secrets/initialAdminPassword

Copy the password and paste it into the Unlock Jenkins page. Jenkins documents this exact Docker-based password retrieval path.

Step 11: Install suggested plugins
Choose:
Install suggested plugins

it installs:-
Git
Pipeline
UI tools

That is the easiest option for first-time setup. Jenkins explicitly offers this during initial setup.

Step 12: Create your first admin user
Add:
username
password
full name
email
Then continue.

Step 13: Save Jenkins URL

Usually:
http://localhost:8080/

Now your Jenkins is ready.

Note:-We run Jenkins inside Docker, and we also give Jenkins its own Docker environment so it can build and deploy applications.
