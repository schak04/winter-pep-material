VT-x (Intel Virtualization Technology):-
VT-x is a CPU feature that enables hardware virtualization or
VT-x is a CPU feature that allows hypervisors to create and run virtual machines efficiently.
It allows a hypervisor (like VirtualBox, VMware, Hyper-V) to create and run virtual machines efficiently.

Need:->Without VT-x enabled, many VM tools either won’t run or will run very slowly.

Enable virtualization is BIOS(Basic Input/Output System)
VT-x must be enabled in BIOS/UEFI so the operating system and hypervisor can use the CPU virtualization features.

Steps:-
Restart computer
Enter BIOS/UEFI (F2 / DEL / ESC depending on system)
Find Virtualization Technology / Intel VT-x
Enable it
Save and restart

Relation Between These Terms
| Term                     | Meaning                                  |
| ------------------------ | ---------------------------------------- |
| **VT-x**                 | CPU feature that supports virtualization |
| **Hypervisor**           | Software that creates and manages VMs    |
| **Virtual Machine (VM)** | A virtual computer running its own OS    |

Networking(Brief):-
If you want to connect your device to any network you need a device called network adaptor.
In mdern days we a wireless adaptor which is connected to wifi router(has an ip address to identify a network device)

A Virtual Machine (VM) has a virtual network adapter (like a network card).
Using Bridged Networking, the VM connects directly to the physical network (router).
The router treats the VM like a separate computer on the same network.

1.No Os installation - it setups vm images/box which are available on vagrant cloud

To install vagrant on local system:-
Vagrant is a commnad line tool not a graphical tool and to execute 
for windows ------------------------> we use  git bash and 
for mac users----------------------->terminal

Here you can follow the commnads to install vagrant on your local device:-
1. look for present working directory.command---->pwd
2. use command mkdir ~/Desktop/vagrant-vms(optional, referred c drive)
3.now go to current folder using command---> cd vagrant-vms
4. create two folders Ex:- centos and ubuntu
5.Go to centos
6. You will be putting a file with command vagrant init and the box name(find the boxname from vagrant cloud) 
follow this link:--> https://portal.cloud.hashicorp.com/vagrant/discover
7. copy eurolinux-vagrant/centos-stream-9 and paste inside gitbash , you check it by typing  command--->ls
8. to know the details of your file use cat Vagrant(if you have done any mistake while writing you can still modify it)
9.You may get some error while installing (common because of antivirous/using any VPN/or corporate netwok)
10.Now follow command---> vagrant up
11. you can check the box list using vagrant box list /vagrant status or vagrant global-status --prune
12. To login follow the command-----> vagrant ssh and to check the user you can follow the command-----> whoami to logout use command---> exit
13.To power off the vm use command----->vagrant halt
14. To reboot the vm use command--->vagrant reload and to remove it usecommand---->vagrant destroy

To check all the boxes installed on your system you can use command---->vagrant box list
