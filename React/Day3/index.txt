Rendering Lists (Arrays) in React
Introduction
Rendering lists, or arrays, in React is a fundamental aspect of building dynamic user interfaces.
It allows developers to display dynamic content, such as user-generated data or information fetched from an external source.
This process is crucial for creating responsive applications that update in real-time.
React simplifies this task through its efficient rendering capabilities, particularly by utilizing the map() function to iterate over array elements and render them as individual components.
This lesson will dive into the techniques for rendering lists in React, highlighting their importance in enhancing application flexibility and interactivity.

 Note:- Importance of key----->
The key prop plays a pivotal role in React's rendering process, especially when dealing with lists.
It provides a unique identifier for each element, enabling React to manage items efficiently during updates.
This mechanism is vital for optimizing performance and ensuring a smooth user experience.
---------------------------------------------------------------
Note:-
key tells React: “This data belongs to this component.”
So React can update only the changed part, not the whole list.

function ToDoList() {
  const [toDoList, setToDoList] = React.useState([
    { id: 1, description: "Item 1" },
    { id: 2, description: "Item 2" },
    { id: 3, description: "Item 3" },
    { id: 4, description: "Item 4" },
  ]);
  return (
    <div>
      <ul>
        {toDoList.map((item) => {
          return <li key={item.id}>{item.description}</li>;
        })}
      </ul>
    </div>
  );
}
function App() {
  return (
    <div className="App">
      <ToDoList />
    </div>
  );
}
const root = createRoot(document.getElementById("root"));
root.render(<App />);
//In this solution, we utilize map() to iterate over the toDoList array and render each item as a list element (<li>). 
The key prop is crucial for helping React identify and track each element in the list, optimizing the rendering process.
